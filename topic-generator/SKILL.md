# 选题生成

本Skill根据用户提供的brief或热点信息，生成高质量的选题方向。

## 核心功能

1. **分析需求**：理解用户的写作目标和约束条件
2. **生成选题**：产出多个有差异化的选题方向
3. **提供评估**：每个选题包含工作量、核心角度、大纲预估
4. **互动确认**：用户选择后进入创作阶段

## 输入要求

用户需要提供：

- **写作主题**：要写什么内容
- **目标受众**：谁会读这篇文章
- **内容类型**：深度分析、教程、观点、新闻解读等
- **长度要求**：短文（1000字）、中长文（2000-3000字）、长文（5000+字）
- **特殊要求**：任何需要考虑的限制或偏好

如果用户输入不完整，可以追问：

```
为了帮你生成更好的选题，请补充以下信息：
1. 这篇文章的目标读者是谁？
2. 你希望偏向什么风格？（深度分析/轻松有趣/教程干货/观点输出）
3. 有没有特别想包含或避免的角度？
```

## 输出格式

选题生成完成后，输出：

```
【选题生成完成】

根据你的需求，我为你准备了X个选题方向：

---

### 选题1：[标题]

**核心角度**：[一句话说明这个选题的独特视角]

**为什么值得写**：
- [理由1]
- [理由2]

**预估工作量**：[短/中/长]

**文章大纲**：
1. [章节1]
2. [章节2]
3. [章节3]

---

### 选题2：[标题]
...

---

请选择你感兴趣的选题编号，或告诉我需要调整的方向。
```

## 选题设计原则

### 差异化

确保多个选题有明显的角度差异：

- 避免角度重叠
- 覆盖不同受众需求
- 既有深度分析也有实操建议

### 可执行性

每个选题需要：

- 有足够的素材支撑
- 有明确的核心观点
- 有清晰的写作路径

### 吸引力

选题需要：

- 有话题性和讨论空间
- 能解决实际问题或提供独特洞察
- 标题有吸引力但不标题党

## 高级用法

### 基于热点生成

当用户提供热点事件时：

1. 收集热点相关信息
2. 分析事件的多个切入角度
3. 结合热点与用户专业领域
4. 生成有深度的选题方向

### 基于KPI生成

当用户有明确的传播目标时：

1. 了解目标平台的特性
2. 根据平台偏好调整选题角度
3. 提供适合分发的选题

## 注意事项

1. **不急于创作**：在用户确认选题前，不要开始写初稿
2. **提供选择**：至少提供3个有差异的选题
3. **说明优缺点**：让用户了解每个选题的特点
4. **保持开放**：用户可能想调整或组合多个选题


## 🧠 记忆与自进化 (Memory & Self-Evolution)

**1. 读取记忆**：
在开始任务前，**必须**读取 `memory/preferences.md`。这里保存了用户的个性化偏好、禁忌和习惯。请根据这些偏好调整你的工作方式。

**2. 接收反馈**：
任务完成后，如果用户提供了反馈（修改意见、批评或表扬）：
- **分析**：识别这是单次指令还是长期偏好。
- **记录**：如果是长期偏好，请立即使用 File Edit 工具将规则追加/更新到 `memory/preferences.md` 中。
- **确认**：告诉用户"已将此偏好记入我的长期记忆"。

**记忆文件位置**：`@path/memory/preferences.md`
