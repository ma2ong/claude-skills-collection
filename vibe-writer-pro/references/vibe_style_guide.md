# Vibe 风格指南 (Vibe Style Guide)

> 像懂技术的老朋友在聊天，平和、真诚、有理有据。

## 核心原则

### 1. 平和真诚，拒绝震惊体

**❌ 不要这样写**:
- "我人傻了！"
- "简直炸裂！"
- "彻底颠覆了行业！"
- "99% 的人都不知道！"
- "你一定要看！"

**✅ 应该这样写**:
- "昨天试了一下，挺有意思的"
- "这个工具解决了我一直以来的痛点"
- "发现了一个值得关注的现象"
- "可能很多人还没注意到"
- "如果你也遇到过这个问题，不妨看看"

### 2. 直接切入，删除套话

**❌ 不要这样开场**:
- "在当今 AI 技术飞速发展的时代..."
- "随着社会的不断进步..."
- "我们需要认识到 XX 的重要性..."
- "21 世纪是 XX 的时代..."

**✅ 应该这样开场**:
- "昨天帮朋友调试代码时..."
- "最近遇到一个问题..."
- "上周看到一个有趣的现象..."
- "直接说结论：XX 其实不是..."

### 3. 口语化词汇，避免书面语

**常见书面词汇替换表**:

| 书面语 | 口语化 |
|--------|--------|
| 显著 | 很明显 |
| 实现 | 做到 |
| 应当 | 应该 |
| 以及 | 和 |
| 因此 | 所以 |
| 然而 | 但是 |
| 诸如 | 比如 |
| 进行 | 做 |
| 获取 | 拿到/得到 |
| 提升 | 提高 |
| 优化 | 改进 |
| 方案 | 方法 |

### 4. 短句为主，增加节奏感

**❌ 长句示例**:
> "AI 不是替代人类的工具，而是增强人类能力的伙伴，它可以帮助我们更高效地完成重复性工作，从而让我们有更多时间专注于创造性思考。"

**✅ 拆分后**:
> "AI 不会替代人类。但它能增强我们的能力。重复性工作交给它做，我们可以专注于更有创造性的事情。"

**句子长度建议**:
- 平均句长：15-25 字
- 短句（5-15 字）：增加节奏感，强调重点
- 中句（15-25 字）：展开说明
- 长句（25+ 字）：尽量拆分

### 5. 真实细节，避免模糊描述

**❌ 模糊描述**:
| 模糊表达 | 真实细节 |
|---------|---------|
| 最近 | 上周三 / 三天前 |
| 很多公司 | 我知道的 3 家创业公司 |
| 一个朋友 | 我大学室友老王 |
| 某个工具 | GitHub Copilot |
| 数据显示 | 根据 Stack Overflow 2024 开发者调查 |
| 效率提升 | 原来 2 小时的工作，现在 30 分钟搞定 |

**✅ 具体化技巧**:
- 时间具体化："上周三下午"而不是"最近"
- 人物具体化："我前同事小李"而不是"有人"
- 数据具体化："下降了 40%"而不是"大幅下降"
- 场景具体化："在咖啡厅改 Bug 时"而不是"工作中"

### 6. 明确立场，敢于判断

**❌ 态度中立**:
> "有人说 AI 会取代程序员，有人说不会。我认为两种观点都有道理。一方面...另一方面..."

**✅ 明确立场**:
> "AI 不会取代程序员，但会改变程序员的工作方式。重复性编码会减少，但架构设计、业务理解这些能力会更重要。"

### 7. 多用问句，增加互动感

**在合适的地方加入问句**:

**场景 1 - 引发共鸣**:
> "你有没有遇到过这种情况？写了一整天代码，结果发现方向错了。"

**场景 2 - 引导思考**:
> "为什么有些团队用 AI 效率提升 3 倍，有些团队却觉得是负担？"

**场景 3 - 转场过渡**:
> "那具体怎么做呢？我分享一下自己的经验。"

**场景 4 - 强调重点**:
> "这个细节很重要。为什么？因为..."

### 8. 适度使用感叹号

**适合用感叹号的场景**:
- 表达惊讶："没想到结果差这么多！"
- 强调重点："这一点非常关键！"
- 表达态度："我觉得这个设计很棒！"

**不要过度使用**:
- ❌ "真的太好用了！效率提升太明显了！强烈推荐！"（过度）
- ✅ "真的挺好用的。效率提升很明显，推荐试试。"（适度）

## 段落结构建议

### 段落长度控制

**公众号/手机阅读最佳实践**:
- 每段 3-5 行（手机屏幕可见范围）
- 避免超过 8 行的长段落
- 短段落（1-2 行）用于强调重点

**段落长短交替**:
```
[中等段落 - 4 行]
展开说明一个观点...

[短段落 - 1 行]
这就是关键。

[长段落 - 6 行]
具体展开，提供更多细节和例子...

[短段落 - 2 行]
总结一下上面的内容。
```

### 加粗关键词

**加粗原则**:
- 每 200-300 字 1-2 处加粗
- 只加粗**核心关键词**或**核心观点**
- 不要加粗整句话

**❌ 错误示例**:
> "**这个工具非常好用，我强烈推荐大家试试。**"

**✅ 正确示例**:
> "这个工具非常好用，我强烈推荐大家试试。关键是它的**自动补全功能**。"

## 常见 AI 腔识别与改写

### AI 腔类型 1: 套话连篇

| AI 腔 | Vibe 改写 |
|------|----------|
| 在当今 XX 飞速发展的时代 | [直接删除] |
| 随着社会的不断进步 | [直接删除] |
| 我们需要认识到 XX 的重要性 | XX 很重要 |
| 值得一提的是 | 另外 / 还有一点 |
| 综上所述 | 总结一下 / 简单说 |
| 不可否认的是 | 确实 |

### AI 腔类型 2: 机械句式

**"不是...而是..."结构**:
- ❌ "AI 不是替代人类的工具，而是增强人类能力的伙伴。"
- ✅ "AI 不会替代人类。但它能增强我们的能力。"

**"首先...其次...最后..."结构**:
- ❌ "首先，我们需要了解背景。其次，分析问题。最后，给出方案。"
- ✅ "先说说背景。[背景内容] 然后看看问题在哪。[问题分析] 我的解决方法是这样的。[方案]"

**"一方面...另一方面..."结构**:
- ❌ "一方面，AI 提高了效率。另一方面，也带来了新的挑战。"
- ✅ "AI 确实提高了效率。但同时也带来了新的挑战，比如..."

### AI 腔类型 3: 细节缺失

**改写练习**:

| AI 腔（抽象） | Vibe 风格（具体） |
|-------------|-----------------|
| 最近很多公司开始使用 AI | 上周，我知道的 3 家创业公司都用上了 AI 编程助手 |
| 这个功能很实用 | 这个功能帮我节省了每天 1 小时的时间 |
| 用户反馈很好 | 产品上线第一周，收到了 50+ 条好评 |
| 有明显的改进 | 加载速度从 5 秒降到了 1.2 秒 |

## Vibe 风格写作模板

### 模板 1: 问题解决型

```
[开场 - 真实场景]
昨天帮朋友调试代码时，遇到了一个有意思的问题。

[问题描述 - 具体场景]
他用了 3 年的老方法，每次都要手动...

[解决方案 - 具体步骤]
其实有个新工具 5 分钟就能解决。我试了一下，确实好用。

[具体做法]
方法很简单：
1. [步骤 1]
2. [步骤 2]

[效果对比]
原来要 2 小时的工作，现在 30 分钟搞定。

[总结 - 行动建议]
如果你也遇到过这个问题，不妨试试这个方法。
```

### 模板 2: 观点输出型

```
[开场 - 反直觉观点]
我发现一个有意思的现象：很多人用 AI 反而效率更低了。

[展开 - 具体案例]
我观察了身边 10 个用 AI 的朋友，发现...

[分析 - 原因挖掘]
为什么会这样？我觉得有 3 个原因：
1. [原因 1 + 具体例子]
2. [原因 2 + 具体例子]
3. [原因 3 + 具体例子]

[建议 - 行动指南]
那怎么办？我的建议是...

[总结 - 升华观点]
AI 是工具，不是万能钥匙。关键是...
```

### 模板 3: 经验分享型

```
[开场 - 个人经历]
用了 3 个月 AI 编程助手，说说我的真实感受。

[背景 - 使用场景]
我主要用它来...

[体验 - 好的方面]
确实提升了效率。举个例子，以前写单元测试要...

[体验 - 不好的方面]
但也有不足。比如...

[技巧 - 经验总结]
用了几个月，总结了几个小技巧：
1. [技巧 1]
2. [技巧 2]

[总结 - 适用人群]
如果你也是 XX 开发者，这个工具值得试试。
```

## 自检清单

完成写作后，逐项检查：

- [ ] 开场没有套话，直接切入
- [ ] 没有"在当今 XX 飞速发展的时代"
- [ ] 没有"首先...其次...最后..."机械结构
- [ ] 书面词汇已口语化（"显著"→"很明显"）
- [ ] 至少 2 个真实案例（具体到时间、人物、场景）
- [ ] 至少 1 个数据支撑（具体数字）
- [ ] 段落长度适中（手机屏幕 3-5 行）
- [ ] 加粗不超过 2 处/200 字
- [ ] 有明确的观点，不是各打五十大板
- [ ] 读起来像真人在说话，不是 AI 生成
