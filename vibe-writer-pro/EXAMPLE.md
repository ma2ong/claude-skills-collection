# Vibe Writer Pro 使用示例

本文档展示了如何使用 Vibe Writer Pro 创建一篇高质量的技术文章。

## 场景示例：撰写一篇关于 AI 编程助手的技术文章

### 第 1 步：启动 Vibe Writer Pro

**用户输入**：
```
"启动 Vibe Writer Pro，我想写一篇关于 AI 编程助手的文章"
```

**Claude 回复**：
```
【需求理解】

为了更好地帮你创作，请补充以下信息：

1. 目标读者是谁？
   - 技术人员（开发者）
   - 产品经理
   - 普通用户

2. 文章目标是什么？
   - 教程（如何使用）
   - 观点（个人看法）
   - 案例分析（实战经验）
   - 新闻解读（行业动态）

3. 预期长度？
   - 短文（1000 字）
   - 中长文（2000-3000 字）
   - 长文（5000+ 字）

4. 发布平台？
   - 公众号
   - 知乎
   - 个人博客
   - 其他

5. 有什么特殊要求吗？
```

### 第 2 步：提供需求信息

**用户输入**：
```
"目标读者是开发者，文章目标是分享我的实战经验，
中长文 2000-3000 字，发布在公众号，希望风格轻松一些"
```

**Claude 回复**：
```
【需求理解】

目标读者：开发者（有一定技术背景）
文章类型：实战经验分享
核心目标：分享 AI 编程助手的真实使用体验和技巧
预估长度：2000-3000 字
发布平台：公众号
特殊要求：轻松风格，Vibe 文风

---

【调研完成】

找到相关资料：15 篇
真实案例：5 个
数据支撑：3 条
个人素材：2 条

核心发现：
1. GitHub Copilot 和 Cursor 是最受欢迎的两个工具
2. 开发者普遍关心"提示词技巧"和"效率提升"
3. 真实的踩坑经验比功能介绍更受欢迎

---

【选题提案】

根据调研，我为你准备了 3 个选题方向：

---

### 选题 1：用了 3 个月 AI 编程助手，说说我的真实感受

**核心角度**：个人真实体验 + 踩坑经验

**为什么值得写**：
- 填补市场空白：多数文章只讲功能，少讲真实体验
- 有充足素材：你的 3 个月使用经历就是最好的素材

**预估工作量**：中

**文章大纲**：
1. 开场：为什么开始用 AI 编程助手
2. 初期体验：惊喜与失望并存
3. 进阶技巧：如何提高效率
4. 真实案例：3 个具体的使用场景
5. 总结：适合谁用，不适合谁用

---

### 选题 2：AI 编程助手的 5 个高效使用技巧

**核心角度**：实操技巧 + 具体方法

**为什么值得写**：
- 解决实际问题：很多人用了 AI 但效率没提升
- 可操作性强：每个技巧都能立即实践

**预估工作量**：短

**文章大纲**：
1. 开场：为什么有人用 AI 反而效率更低
2. 技巧 1：[具体技巧]
3. 技巧 2：[具体技巧]
4. 技巧 3：[具体技巧]
5. 技巧 4：[具体技巧]
6. 技巧 5：[具体技巧]
7. 总结：从工具到习惯

---

### 选题 3：AI 编程助手如何改变我的开发流程

**核心角度**：工作流变化 + 深度思考

**为什么值得写**：
- 独特视角：从工作流角度看 AI 的影响
- 有讨论空间：引发读者思考自己的工作流

**预估工作量**：长

**文章大纲**：
1. 开场：3 个月前的开发流程 vs 现在的流程
2. 变化 1：需求理解阶段
3. 变化 2：代码编写阶段
4. 变化 3：调试测试阶段
5. 反思：AI 是工具还是伙伴
6. 总结：未来的开发会是什么样

---

请选择你感兴趣的选题编号（1/2/3），或告诉我需要调整的方向。
```
